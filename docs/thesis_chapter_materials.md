本文档整理毕业论文中“系统设计、系统实现、系统测试”三章可直接改写使用的内容。正式写入论文时，可根据学校模板调整标题层级、图表编号和行文格式。

## 第 4 章 系统设计

### 4.1 系统总体设计

本系统面向疲劳驾驶场景中的驾驶员面部状态识别任务，采用“目标检测模型 + 疲劳状态判定 + 桌面演示系统”的总体架构。系统首先通过摄像头、视频或图片输入获取驾驶员面部图像，然后使用 YOLOv8 系列目标检测模型识别闭眼、睁眼和打哈欠等疲劳相关面部特征，最后结合滑动窗口规则对驾驶员疲劳状态进行判断，并在桌面界面中实时显示检测结果、疲劳状态和报警信息。

系统整体流程如下：

```text
图像/视频输入 -> 图像预处理 -> YOLOv8 检测 -> 疲劳特征提取 -> 疲劳状态判断 -> 界面显示与日志保存
```

本系统主要包括四个功能模块：

| 模块 | 主要功能 |
| --- | --- |
| 数据管理模块 | 管理 YOLOv8 格式数据集、训练配置和实验结果 |
| 模型训练模块 | 完成 YOLOv8n 基线模型与 YOLOv8n + CBAM 注意力模型训练 |
| 推理检测模块 | 支持图片、视频和摄像头输入，输出检测框、类别和置信度 |
| 桌面演示模块 | 显示检测画面、疲劳状态、报警信息，并支持截图和日志保存 |

### 4.2 数据集设计

本研究使用 Roboflow 平台整理的 Driver_Drowsiness_YOLO 数据集。该数据集包含 5413 张图像，并已按照 YOLOv8 所需格式划分为训练集、验证集和测试集。

| 数据划分 | 图片数量 | 标签数量 |
| --- | ---: | ---: |
| 训练集 | 3790 | 3790 |
| 验证集 | 542 | 542 |
| 测试集 | 1081 | 1081 |
| 合计 | 5413 | 5413 |

检测类别包括闭眼、睁眼和打哈欠三类，类别顺序如下：

| 类别编号 | 类别名称 | 中文含义 |
| ---: | --- | --- |
| 0 | Eyeclosed | 闭眼 |
| 1 | Eyeopen | 睁眼 |
| 2 | Yawn | 打哈欠 |

数据集配置文件为 `configs/yolo_data.yaml`。训练过程中使用 Mosaic、MixUp 和 HSV 色彩扰动等增强策略，以提升模型对不同光照、姿态和局部遮挡场景的适应能力。

### 4.3 模型结构设计

本研究首先使用 YOLOv8n 作为基线模型。YOLOv8n 具有参数量小、推理速度快和部署成本低等特点，适合车载终端和普通 PC 环境下的实时检测任务。

为体现注意力机制对疲劳特征识别的优化作用，本研究进一步设计 YOLOv8n + CBAM 模型。CBAM 注意力模块包含通道注意力和空间注意力两部分，能够在特征提取过程中增强模型对关键通道和关键空间区域的关注能力。考虑到闭眼和打哈欠均属于局部面部特征，注意力机制有助于提升模型对眼部和嘴部区域的辨别能力。

本项目将 CBAM 模块加入 YOLOv8n 主干网络 SPPF 模块之后，配置文件为 `configs/yolov8n_cbam.yaml`。该位置位于主干网络高层语义特征输出处，有利于在进入检测头前强化疲劳相关特征表达。

### 4.4 疲劳状态判定设计

YOLOv8 检测模型输出每帧中的 `Eyeclosed`、`Eyeopen` 和 `Yawn` 检测结果。系统将每帧检测结果转换为疲劳特征，包括闭眼置信度、睁眼置信度和打哈欠置信度。随后，系统通过滑动窗口规则进行疲劳状态判断。

当前系统状态分为三类：

| 状态 | 判定含义 |
| --- | --- |
| 正常 | 未出现明显闭眼或打哈欠特征 |
| 疑似疲劳 | 出现一次打哈欠，或闭眼比例达到疑似阈值 |
| 疲劳 | 闭眼比例较高，或短时间内多次出现打哈欠 |

该规则具有解释性强、实现简单和便于演示的优点。后续若进一步扩展，可使用 LSTM + Attention 模型对连续帧特征进行时序建模，从而进一步提升疲劳状态判断的稳定性。

### 4.5 系统界面设计

桌面演示系统使用 PyQt5 实现，主要支持以下功能：

- 选择本地图片进行单张检测；
- 选择本地视频进行连续检测；
- 打开摄像头进行实时检测；
- 显示检测框、类别名称和置信度；
- 显示正常、疑似疲劳、疲劳三类状态；
- 疲劳状态使用红色文字进行提示；
- 保存当前检测截图；
- 自动生成 CSV 检测日志。

## 第 5 章 系统实现

### 5.1 开发环境实现

本项目采用 Python 作为主要开发语言，深度学习框架使用 PyTorch，目标检测模型基于 Ultralytics YOLOv8 实现，图像处理使用 OpenCV，桌面界面使用 PyQt5。服务器训练环境使用 Slurm 调度系统和 Tesla V100 GPU，本地演示环境使用 Windows CPU 推理。

项目主要目录如下：

| 路径 | 说明 |
| --- | --- |
| `configs/` | 数据集配置和模型配置 |
| `src/train/` | YOLOv8 训练入口 |
| `src/infer/` | 图片、视频、摄像头推理入口 |
| `src/app/` | PyQt5 桌面演示系统 |
| `src/utils/` | 疲劳规则和 Ultralytics 兼容补丁 |
| `scripts/slurm/` | Slurm GPU 训练脚本 |
| `docs/` | 实验记录、系统测试和论文材料 |
| `weights/` | 训练后的核心模型权重 |
| `runs/` | 训练结果和曲线图 |

### 5.2 模型训练实现

模型训练入口为 `src/train/train_yolo.py`。该脚本封装了 Ultralytics YOLO 的训练接口，支持指定数据集配置、模型配置、训练轮数、输入尺寸、batch size、worker 数量和设备编号。对于注意力机制模型，脚本增加了 `--pretrained-weights` 参数，用于从 YOLOv8n 预训练权重迁移初始化。

基线模型训练脚本为：

```bash
sbatch scripts/slurm/train_yolo.sbatch
```

CBAM 注意力模型训练脚本为：

```bash
sbatch scripts/slurm/train_yolo_cbam.sbatch
```

训练过程中使用 AdamW 优化器和 Cosine LR 学习率策略，并启用 Mosaic、MixUp 和 HSV 色彩扰动增强。

### 5.3 注意力机制实现

Ultralytics 当前版本中已包含 CBAM 模块，但部分版本在解析自定义 YAML 时未将 CBAM 注册到模型解析器命名空间中。因此，项目在 `src/utils/ultralytics_patches.py` 中实现了兼容注册函数 `register_attention_modules()`，在训练、推理和桌面系统加载模型前显式注册 CBAM、ChannelAttention 和 SpatialAttention 模块。

该做法不修改第三方库源码，便于在本地和服务器环境中保持一致。

### 5.4 推理检测实现

命令行推理入口为 `src/infer/run_infer.py`，支持图片、视频和摄像头三类输入。推理流程包括：

1. 加载 YOLOv8 权重；
2. 读取输入图像或视频帧；
3. 执行模型预测；
4. 提取检测类别和置信度；
5. 计算疲劳状态；
6. 绘制检测框、状态和 FPS；
7. 按需保存检测结果。

示例命令：

```powershell
.venv\Scripts\python.exe -m src.infer.run_infer --weights weights/best_cbam.pt --source data\fatigue_yolo\test\images\0_jpg.rf.8f6a5837638646b3083c01fd0938d5f6.jpg --save runs\demo_cbam.jpg
```

### 5.5 桌面系统实现

桌面系统入口为 `src/app/main_window.py`。界面使用独立检测线程 `VideoWorker` 执行模型推理，避免主界面卡顿。检测线程负责读取图片、视频或摄像头帧，并将检测后的图像、状态文本和日志路径发送给主界面。

系统检测日志保存至 `runs/app_logs/`，截图保存至 `runs/screenshots/`。正式文档中使用的系统截图已复制到 `docs/images/`。

## 第 6 章 系统测试与实验分析

### 6.1 模型训练结果

YOLOv8n 基线模型和 YOLOv8n + CBAM 注意力模型均训练 50 轮。两组实验在相同数据集和相同输入尺寸下进行。

| 模型 | Precision | Recall | mAP50 | mAP50-95 | 参数量 | 推理耗时 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| YOLOv8n | 0.6868 | 0.9114 | 0.8347 | 0.5368 | 约 3.01M | 0.9 ms |
| YOLOv8n + CBAM | 0.6840 | 0.9370 | 0.8580 | 0.5470 | 约 3.08M | 1.1 ms |

实验结果表明，引入 CBAM 注意力机制后，模型 mAP50 从 0.8347 提升至 0.8580，mAP50-95 从 0.5368 提升至 0.5470，Recall 从 0.9114 提升至 0.9370。虽然模型参数量和推理耗时略有增加，但仍满足实时检测需求。

### 6.2 类别检测结果分析

CBAM 模型最终验证指标如下：

| 类别 | Images | Instances | Precision | Recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Eyeclosed | 278 | 353 | 0.871 | 0.946 | 0.973 | 0.574 |
| Eyeopen | 107 | 144 | 0.657 | 0.889 | 0.839 | 0.531 |
| Yawn | 81 | 81 | 0.524 | 0.975 | 0.762 | 0.535 |

其中，闭眼类别识别效果最好，mAP50 达到 0.973；打哈欠类别 mAP50 达到 0.762，相比基线模型有明显提升，说明注意力机制对嘴部局部特征具有一定增强作用。

### 6.4 实验结论

本阶段实验表明，系统能够完成驾驶员面部疲劳相关目标检测，并可通过桌面系统进行可视化展示。YOLOv8n + CBAM 模型在整体 mAP 和 Recall 上优于 YOLOv8n 基线模型，能够作为论文中注意力机制改进的主要实验依据。
