# 系统测试记录

## 1. 测试目标

本测试用于验证疲劳驾驶面部识别系统在本地 Windows 环境下是否具备可演示能力，包括模型加载、图片检测、疲劳状态判断、检测日志生成和截图保存功能。

## 2. 测试环境

| 项目 | 配置 |
| --- | --- |
| 系统形态 | PyQt5 桌面系统 |
| 模型权重 | `weights/best.pt` |
| 推理设备 | CPU |
| PyTorch | 2.5.1+cpu |
| 输入类型 | 图片 |

## 3. 功能测试

| 测试项 | 测试方法 | 预期结果 | 当前状态 |
| --- | --- | --- | --- |
| 模型加载 | 启动桌面系统并加载 `weights/best.pt` | 模型正常加载，无 DLL 报错 | 通过 |
| 图片检测 | 选择测试集图片并点击开始检测 | 显示检测框、类别和置信度 | 通过 |
| 疲劳状态显示 | 输入打哈欠图片 | 显示疑似疲劳或疲劳状态 | 通过 |
| 日志记录 | 完成一次检测 | 在 `runs/app_logs/` 生成 CSV 日志 | 通过 |
| 截图保存 | 点击保存截图 | 在 `runs/screenshots/` 保存检测画面 | 待保存正式截图 |
| 摄像头输入 | 点击打开摄像头 | 摄像头可用时实时显示检测结果 | 本机摄像头暂不可用 |

## 5. 测试结论

系统已完成本地演示闭环，能够在无摄像头、无视频样例的情况下，通过测试图片展示模型检测效果。当前演示功能已满足毕业设计答辩展示的基础要求。

正式答辩前建议补充：

- 保存一张清晰的系统截图到 `runs/screenshots/`
- 准备 3 至 5 张不同状态的测试图片，包括闭眼、睁眼、打哈欠
- 如能获取摄像头或手机录制视频，可增加一段视频演示材料
